最短路三兄弟：Dijkstra、Bellman-Ford、Floyd

发表于2026-03-09|更新于2026-03-05|算法
|总字数:2.1k|阅读时长:7分钟|浏览量:

图论题里最高频的考点之一。很多人三个算法都会背，但上了真题分不清楚该用哪个，或者想当然地用了错的那个然后WA到怀疑人生。这篇文章的目标是把三个算法的适用边界说清楚，不只是”会背”。

Dijkstra：贪心 + 优先队列

基本思路

Dijkstra 解决的是单源最短路（Single Source Shortest Path, SSSP）问题：给定源点 $s$，求 $s$ 到图中所有其他点的最短距离。

算法核心是贪心：维护一个集合 $S$，初始只含源点 $s$，$dist[s] = 0$，其他点 $dist[v] = +\infty$。每次从未确定的点里找 $dist$ 最小的点 $u$，把 $u$ 加入 $S$，然后用 $u$ 的出边对邻居做松弛（Relaxation）操作：

$$\text{if } dist[u] + w(u, v) < dist[v]: \quad dist[v] \leftarrow dist[u] + w(u, v)$$

这个贪心选择的正确性依赖一个前提：边权非负。

用御坂美琴找最省电路线做类比：她每次选择”当前已知路径里，到达费电最少的那个节点”去探索，因为电量不能是负数，一旦找到某条路是当前最省电的，就不可能有更省电的路被后来发现。如果某条线路可以”充电”（负权边），这个贪心选择就会失效——你以为当前最省电的路就是最优的，但走了一圈充了电之后发现另一条路更好。

朴素 Dijkstra 是 $O(V^2)$，用优先队列（二叉堆）优化后是 $O((V+E)\log V)$，用 Fibonacci 堆是 $O(E + V \log V)$（理论最优但实现复杂，竞赛里不用）。

Bellman-Ford：动态松弛，能检测负环

基本思路

Bellman-Ford 的思路不是贪心，而是重复松弛。核心定理：在一个 $V$ 个节点的图里，如果存在最短路，最短路最多经过 $V-1$ 条边。所以做 $V-1$ 轮松弛，每轮对所有边做一次松弛操作，就能保证找到所有最短路。

时间复杂度 $O(VE)$，比 Dijkstra 慢，但可以处理负权边，而且可以检测负环：做完 $V-1$ 轮之后再做第 $V$ 轮，如果还有边能被松弛，说明图中存在负环（因为负环可以让路径长度无限减小）。

SPFA：Bellman-Ford 的队列优化，以及它被 hack 的历史

SPFA（Shortest Path Faster Algorithm）是 Bellman-Ford 的队列优化版本：把”已经被松弛过的点”放进队列，下一轮只处理队列里的点而不是全部边，平均情况下复杂度接近 $O(kE)$（$k$ 是一个较小的常数）。

在 2010 年代之前的国内竞赛里，SPFA 几乎是单源最短路的万金油写法——比 Dijkstra 好写，还能处理负权边，平均速度也不慢。

然而 SPFA 的最坏情况复杂度还是 $O(VE)$，和普通 Bellman-Ford 一样。出题人发现这一点之后开始专门构造菊花图（一个中心节点连接大量叶子节点，再绕回来）来卡 SPFA，导致 SPFA 频繁 TLE。现在稍微正式一点的比赛，如果 checker 专门卡 SPFA，这个算法就是 TLE 的等价物。

现在的竞赛标准是：能用 Dijkstra 就用 Dijkstra，不能用（有负权边但没有负环）再考虑 Bellman-Ford/SPFA。用 SPFA 之前先看看数据范围和题目来源，如果是近年的 Codeforces 题或者有”卡 SPFA”声誉的出题人，趁早换。

Floyd-Warshall：全源最短路，O(V³) 的 DP

基本思路

Floyd 解决的是全源最短路（All-Pairs Shortest Path, APSP）：求图中所有点对 $(i, j)$ 之间的最短距离。

算法是 DP。定义 $dp[k][i][j]$ 为”只允许通过节点集合 ${1, 2, \ldots, k}$ 中的节点作为中间节点时，从 $i$ 到 $j$ 的最短距离”。状态转移：

$$dp[k][i][j] = \min(dp[k-1][i][j],\ dp[k-1][i][k] + dp[k-1][k][j])$$

初始状态 $dp[0][i][j] = w(i,j)$（直接相连）或 $+\infty$（不相连），$dp[0][i][i] = 0$。

由于 $dp[k]$ 只依赖 $dp[k-1]$，可以用滚动数组压缩到 $O(V^2)$ 空间，也就是常见的三重循环写法：

注意 if (dp[i][k] != INF && dp[k][j] != INF) 这个判断，防止 INF + INF 溢出成负数导致错误。

Floyd 可以处理负权边，但不能有负环（负环上的 $dp[i][i]$ 会被更新成负数，可以用 $dp[i][i] < 0$ 来检测负环是否存在）。

$O(V^3)$ 的复杂度，$V \leq 500$ 是上限，超过就 TLE。

三者对比

算法 时间复杂度 空间复杂度 负权边 负环检测 用途
Dijkstra（堆优化） $O((V+E)\log V)$ $O(V+E)$ 不支持 不支持 单源，无负权
Bellman-Ford $O(VE)$ $O(V)$ 支持 支持 单源，有负权/需检测负环
SPFA $O(kE)$ 均摊，最坏 $O(VE)$ $O(V)$ 支持 支持 慎用，容易被卡
Floyd $O(V^3)$ $O(V^2)$ 支持 可检测 全源，$V \leq 500$

竞赛选择策略

  1. 题目问单源最短路，边权全为非负 → 优先队列 Dijkstra，不用想
  2. 题目有负权边但 $V$ 不大（$\leq 10^4$） → Bellman-Ford
  3. 题目需要检测负环 → Bellman-Ford 第 $V$ 轮检测，或 Floyd 看对角线
  4. 题目要求所有点对之间的最短距离，$V \leq 500$ → Floyd
  5. 看到”SPFA可以做这题” → 想想会不会被卡，如果不确定，改写 Dijkstra

一个经验：很多题貌似有负权边，但仔细读题发现是差分约束或者边权可以转化，最终还是能用 Dijkstra。读题时要确认到底是”边权真的为负”还是”只是题面描述看起来有减法”。

做题 Checklist

  1. 单源还是多源？多源可以建虚拟超级源点，连到所有真实源点，距离设为0，转化成单源
  2. 有负权边吗？有的话 Dijkstra 直接排除
  3. 需要检测负环吗？需要的话用 Bellman-Ford 第 $V$ 轮，或者用 Floyd 之后看 $dp[i][i]$
  4. 图的规模？$V \leq 500$ 可以考虑 Floyd；$V, E$ 很大 → 必须 Dijkstra
  5. dist 数组初始化成 INF 了吗？没有就全局 bug
  6. Floyd 里三重循环的顺序对了吗？必须是 k 在最外层，i、j 在内层。顺序错了结果是错的而且很难发现

第6条是 Floyd 的经典坑，k 在内层时相当于没有完成”允许通过前 k 个节点”的递推，答案会偏大。
