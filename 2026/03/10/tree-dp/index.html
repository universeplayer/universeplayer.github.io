树形DP：当动态规划遇上树结构

发表于2026-03-10|更新于2026-03-05|算法
|总字数:1.2k|阅读时长:4分钟|浏览量:

树形 DP 是竞赛里的经典题型，看起来变化多端，但核心就一句话：在树上做后序遍历，用子节点的状态推父节点的状态。理解了这一点，大多数树形 DP 题就只是状态设计的问题了。

从一个类比开始

芙宁娜的审判体系是三层的：陪审团评议 → 法庭裁决 → 最终判决。下级的结果汇总之后才能做出上层的裁决，任何一级都不能越级直接影响最终结果。树形 DP 的逻辑和这个完全一样：叶节点先算，然后一层层往上汇总，根节点的答案最后得出。这就是后序遍历的本质。

经典问题1：树上最大独立集

给定一棵 $n$ 个节点的带权树，选一个点集使得总权重最大，且集合中任意两个点不相邻（即不存在直接连边）。

状态定义：

$$dp[u][0] = \text{不选节点 } u \text{ 时，} u \text{ 子树内的最大权重}$$

$$dp[u][1] = \text{选节点 } u \text{ 时，} u \text{ 子树内的最大权重}$$

转移方程：

$$dp[u][1] = w_u + \sum_{v \in \text{children}(u)} dp[v][0]$$

$$dp[u][0] = \sum_{v \in \text{children}(u)} \max(dp[v][0],\ dp[v][1])$$

逻辑很直接：选了 $u$，它的所有儿子就不能选；不选 $u$，每个儿子可以自由选择最优的。

经典问题2：树的直径

树的直径是树中最长的路径（边权之和最大的简单路径）。

一种做法是两次 BFS/DFS，但树形 DP 的做法更直接：在 DFS 过程中，对每个节点 $u$，维护从 $u$ 出发往下走的最长链 $\text{dep}[u]$，经过 $u$ 的最长路径就是两个儿子最长链之和。

$$\text{ans} = \max_{u}\ \max_{v_1 \neq v_2 \in \text{children}(u)} (\text{dep}[v_1] + w(u,v_1) + \text{dep}[v_2] + w(u,v_2))$$

实现时只需要在 DFS 里维护当前节点的最长和次长链，更新全局答案。

换根 DP

换根 DP 是树形 DP 的进阶，用来解决”以每个节点为根，求某个值”的问题。

朴素做法是以每个节点为根分别 DFS，复杂度 $O(n^2)$，无法接受。换根 DP 的做法是两次 DFS：

  • 第一次 DFS（下行）：以节点 1 为根，计算每个节点往子树方向的信息
  • 第二次 DFS（上行）：将根从父节点”传递”给子节点，利用已有信息 $O(1)$ 更新

以”每个节点到其余所有节点的距离之和”为例：

$$\text{sum}[u] = \text{subtree_sum}[u] + \text{up_sum}[u]$$

其中 $\text{subtree_sum}[u]$ 是 $u$ 往子树方向的贡献，$\text{up_sum}[u]$ 是 $u$ 往父节点方向的贡献。

当根从 $u$ 换到其儿子 $v$ 时，$v$ 子树里的所有节点距离减少 1，$v$ 子树外的所有节点距离增加 1：

$$\text{up_sum}[v] = \text{up_sum}[u] + \text{subtree_sum}[u] - \text{subtree_sum}[v] + (n - \text{size}[v])$$

换根 DP 的关键是找到父子之间状态转移的增量关系，一旦推出这个公式，整个算法就是两次线性 DFS，复杂度 $O(n)$。

做题 Checklist

树形 DP 有几个容易错的地方，每次做题可以对照检查：

  1. 状态定义够不够清晰？”以 $u$ 为根的子树”是标准的状态范围，不要混入子树外的信息
  2. 转移顺序对不对？必须是后序（先处理子节点，再处理当前节点），用递归自动保证
  3. 换根时有没有重复计算？从 $u$ 推 $v$ 时，要把 $v$ 原本对 $u$ 的贡献减掉
  4. 树的根是否影响答案？如果不影响，任取一个节点作根；如果影响，需要换根
  5. 有没有处理根节点的边界？根节点没有父节点，初始化时要注意

树形 DP 本质上只是”把 DP 的数组换成树节点”，难点都在状态设计和转移的推导，而不是代码实现。一旦方程推出来，代码基本是模板。多做几道题，自然就有感觉了。

如果你做过芙宁娜相关的审判线剧情，你会发现每一级证据链都需要下级充分证明才能往上推进——这和树形 DP 的逻辑几乎是同构的。当然，游戏里允许情感 override 逻辑，算法里不行。
