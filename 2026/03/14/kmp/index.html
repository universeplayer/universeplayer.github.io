KMP：字符串匹配界的Dijkstra

发表于2026-03-14|更新于2026-03-05|算法
|总字数:1.8k|阅读时长:6分钟|浏览量:

字符串匹配是个听起来很简单的问题：给你一个文本串 s，一个模式串 p，问 p 在 s 里哪里出现过。

朴素解法显而易见：枚举 s 的每个位置，试着从那里开始匹配 p，失败了就从下一个位置重新来。时间复杂度 O(nm)，n 是文本串长度，m 是模式串长度。

朴素解法不是不对，是浪费了。每次失配之后，你丢掉了所有已经匹配过的信息，从头再来。这是典型的”做了重复工作”的问题。KMP 的核心思想就是：失配时不要扔掉那些信息，利用它跳过不必要的比较。

这个算法由 Knuth、Morris 和 Pratt 在 1977 年正式发表 [Knuth, Morris, Pratt, “Fast Pattern Matching in Strings”, SIAM Journal on Computing, 1977]。Knuth 后来在 TAOCP 里说，这是他见过的最优雅的线性时间算法之一。

失配时到底发生了什么

假设 s = "abababc"，p = "ababc"。

我们从 s[0] 开始匹配 p：

  • s[0..3] = "abab" 匹配了 p[0..3] = "abab"
  • s[4] = 'a' vs p[4] = 'c'，失配

朴素算法：从 s[1] 重新开始。但 s[1] = 'b'，p[0] = 'a'，一比就不行，再从 s[2] 开始。

等一下。我们已经知道 s[0..3] = "abab"，所以 s[2..3] = "ab" = p[0..1]。也就是说，我们可以直接把 p 滑到 s[2] 的位置，从 p[2] 继续比，而不是从头开始。已经匹配的 "ab" 不需要再比一次。

这就是 KMP 的精髓：失配时，主串指针 i 不回退，只移动模式串指针 j，跳回到某个更早的位置继续比。跳到哪里，由”前缀函数”告诉你。

前缀函数（failure function）

前缀函数 $\pi[i]$ 的定义：对于模式串 p[0..i]，最长的”真前缀 = 真后缀”的长度。

真前缀是除了整个字符串本身的前缀，真后缀同理。

以 p = "ababc" 为例：

  • p[0] = "a"：没有真前缀，$\pi[0] = 0$
  • p[0..1] = "ab"：真前缀 "a"，真后缀 "b"，不等，$\pi[1] = 0$
  • p[0..2] = "aba"：真前缀 {"a", "ab"}，真后缀 {"a", "ba"}，公共部分是 "a"，$\pi[2] = 1$
  • p[0..3] = "abab"：公共最长是 "ab"，$\pi[3] = 2$
  • p[0..4] = "ababc"：没有公共部分，$\pi[4] = 0$

所以 $\pi = [0, 0, 1, 2, 0]$。

失配时，如果 s[i] != p[j]，把 j 更新为 $\pi[j-1]$，继续比较 s[i] 和 p[j]。这等价于：模式串已经匹配的 p[0..j-1] 里，它的最长公共前后缀就是下次可以直接继承的”已知信息”。

应用：字符串的最小周期

KMP 有一个很好看的推论，涉及到字符串的周期性。

定义：如果字符串 s 满足 s[i] = s[i mod T] 对所有 i 成立，则 T 是 s 的一个周期。

定理：长度为 m 的字符串 s，$m - \pi[m-1]$ 是 s 的最小周期，当且仅当 m 能被 $m - \pi[m-1]$ 整除。[Knuth, Morris, Pratt, 1977，推论]

举例：p = "abababab"，$\pi[7] = 6$，所以最小周期候选是 $8 - 6 = 2$，确实 "ab" 重复 4 次，整除成立，最小周期是 2。

这就是初音未来的”循环采样”——她的 VOCALOID 合成里大量使用周期性的音素，一段音频如果能被分解成最短的循环单元，存储和处理的成本就大幅下降。字符串的最小周期，本质上是在问”这段序列可以被压缩成多短的重复单元”，和音频的周期性分析在概念上完全对应。

next 数组 vs π 数组：一个会踩的坑

很多教材（包括部分 OI 教材和国内竞赛圈）用的是 next 数组而不是 $\pi$ 数组，但两者的定义不一样：

  • $\pi[i]$：p[0..i] 最长公共真前后缀的长度，范围 [0, i]
  • next[i]（某些定义）：p[0..i-1] 最长公共真前后缀的长度，下标偏移了 1
  • next[j]（另一些定义）：失配时 j 应该跳到的位置，等于 $\pi[j-1]$，且通常 next[0] = -1

这三种定义在代码里差了一个下标，混了就是 WA 或者 RE。

建议：竞赛时选定一种写法（我用上面的 $\pi$ 定义），确认完全理解后只用这一种，不要在比赛中临时换定义。

做题 Checklist

拿到字符串匹配题，判断是否需要 KMP：

  1. 需要找模式串在文本串中的所有出现位置→KMP
  2. 需要找最长公共前后缀→直接利用 $\pi$ 数组
  3. 需要最小周期→$m - \pi[m-1]$，注意整除条件
  4. 多模式匹配→考虑 Aho-Corasick 自动机（KMP 的扩展）
  5. 只需要判断是否存在→KMP 也行，但 C++ STL 的 string::find() 也够用（底层可能是不同算法，但不用手写）

最后：π 数组的计算过程本质上是让模式串和自己做一次 KMP。理解了这一点，$\pi$ 的计算和主匹配的代码结构完全一致，只需要记一套逻辑。

KMP 好在什么地方？不是它快——O(nm) 对大多数实际数据已经够了。好在它是个干净的算法，利用了问题结构的所有信息，没有浪费任何一次比较的结果。这种感觉在算法里不多见。
