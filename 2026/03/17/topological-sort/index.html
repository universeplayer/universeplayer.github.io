拓扑排序：有向无环图的线性化

发表于2026-03-17|更新于2026-03-05|算法
|总字数:1.9k|阅读时长:6分钟|浏览量:

枫丹审判庭有一条不成文的规则：轻罪先判，重罪后判。毕竟你不能在一个人还没被定罪的情况下就对他施加依赖于定罪结果的附加刑罚——后面的判决依赖前面的判决。

这就是拓扑排序在做的事：给一堆有依赖关系的节点找一个合法的线性顺序，让每条”A必须在B前面”的约束都得到满足。

问题定义

给定一个有向无环图（Directed Acyclic Graph，DAG），找一个节点的线性序列，使得对图中每条有向边 $u \rightarrow v$，$u$ 在序列中都出现在 $v$ 前面。

这样的序列叫做拓扑序（Topological Order）。

两个前提条件缺一不可：

有向：边有方向，$u \rightarrow v$ 不等于 $v \rightarrow u$。

无环：如果图里有环（$A \rightarrow B \rightarrow C \rightarrow A$），拓扑序不存在——三个节点互相依赖，谁也没法排在最前面。

现实中的典型场景：课程选修（先修 A 才能选 B）、任务调度（A 完成之后才能开始 B）、编译依赖（编译 A 需要先编译它依赖的库）。

Kahn算法（BFS思路）

Kahn算法的直觉非常自然：没有任何依赖的节点（入度为0的节点）可以最先处理。处理完一个节点后，它的所有后继节点的依赖数减一，如果某个后继的依赖数减为0，它也可以处理了。

算法步骤：

  1. 计算所有节点的入度。
  2. 把所有入度为0的节点加入队列。
  3. 循环：取出队列头节点，加入结果序列，对它的所有邻居入度减1，如果某邻居入度变为0，加入队列。
  4. 循环结束，如果结果序列包含所有节点，拓扑排序成功；如果结果序列长度 < 节点总数，说明图中有环。

时间复杂度 $O(V + E)$，空间复杂度 $O(V)$。

DFS实现

DFS的拓扑排序思路是：后序遍历（postorder traversal）的逆序就是拓扑序。

直觉：DFS访问一个节点时，先递归访问它的所有后继，后继都处理完之后，当前节点才入栈。最后把栈里的顺序反转，就是拓扑序。

三色标记是关键：白色（未访问）、灰色（访问中，在当前DFS路径上）、黑色（已完成）。如果DFS从一个灰色节点走到另一个灰色节点，说明存在环。

检测环的方式

两种算法都自带环检测：

Kahn算法：如果最后 order.size() != n，说明有节点的入度始终没降到0，即存在环。

DFS算法：如果访问到颜色为”访问中”（灰色）的节点，说明当前路径形成了环。

实战中，Kahn算法的环检测更容易理解，不容易搞错；DFS的实现更紧凑，但三色标记要写对。

DAG上的最短/最长路

这是拓扑排序最有价值的应用之一。

在一般有向图上，最短路需要Dijkstra（$O((V+E)\log V)$，要求边权非负）或Bellman-Ford（$O(VE)$，允许负权）。但在DAG上，按拓扑序处理节点，可以用DP在 $O(V+E)$ 内解决最短/最长路问题。

最长路只需要把 min 改成 max，初始值改成 INT_MIN。一般有向图上最长路是NP-hard问题（旅行商问题的特例），但DAG上是多项式时间可解的。

关键路径（Critical Path Method）

工程项目管理里有一个经典方法叫关键路径法（CPM）。把项目的各个任务建模成DAG，每个节点是任务，边表示依赖关系，边权是任务持续时间，问”整个项目最少需要多久”。

答案就是DAG上的最长路。关键路径是从起点到终点的所有路径里最长的那条——这条路径上的任何任务延误都会直接影响项目总工期。关键路径外的任务有一定的float（浮动时间），延误不一定影响总工期。

这就是为什么同样的编译依赖，某些模块的优化能加速整体构建，某些模块的优化根本没效果——取决于它在不在关键路径上。

经典题目

LeetCode 207（课程表）：给定若干课程依赖关系，判断能否完成所有课程（等价于判断有向图是否有环）。

LeetCode 210（课程表II）：输出一个合法的学习顺序（就是拓扑排序本身）。

这两道题是Kahn算法的直接应用，建议先拿这两道练手。

做题Checklist

  1. 图是否保证是DAG？如果不保证，要先用环检测处理有环的情况。
  2. 是Kahn（BFS）还是DFS实现？两者都行，但DFS版要把三色标记写对，不要用bool visited（两色不够）。
  3. 拓扑序是否唯一？不唯一——入度为0的节点可能有多个，选哪个先处理都合法，但结果序列可能不同。如果题目要求字典序最小，把队列换成优先队列。
  4. DAG DP时，是先求拓扑序再DP，还是在Kahn过程中直接更新DP值？两种写法都行，后者稍微更紧凑。
  5. 节点编号从0开始还是1开始？别搞错边界。

回到枫丹审判庭。

芙宁娜的判决需要依赖先前所有相关证据的认定，证据认定依赖证人证词，证人证词依赖传唤程序……整个司法体系是一个精心设计的DAG，每个节点都有它的前置依赖。

能保证这套体系不出死锁（环）的方式，就是确保没有任何两个程序互为前置依赖——这在法律设计层面是一个拓扑性质的约束。

照这样看，写过拓扑排序的人，其实对法律逻辑的底层结构有一点点直觉。尽管枫丹的法律是虚构的。
