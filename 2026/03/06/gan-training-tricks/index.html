GAN训练为什么这么难：从理论到工程的血泪

发表于2026-03-06|更新于2026-03-04|技术
|总字数:1.6k|阅读时长:5分钟|浏览量:

做完MTSIR3-GAN这个毕设之后，我对”GAN很难训练”这句话有了非常具体的理解——不是”有点麻烦”那种难，是那种loss曲线看起来一切正常、模型却在悄悄摆烂的难。这篇文章写给同样在和GAN死磕的人，从数学原因到工程解法，尽量不废话。

问题出在哪里：这不是一个优化问题

普通的深度学习训练是找一个损失函数的最小值，梯度下降，结束。GAN不是。

GAN的本质是一个 minimax 博弈：

$$\min_G \max_D ; \mathbb{E}{x \sim p\text{data}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]$$

生成器 G 想骗过判别器 D，判别器 D 想识破生成器。两个网络同时在优化，互为对手。这个结构没有单一的”收敛目标”——你在找的不是一个最小值点，而是一个纳什均衡。

纳什均衡不保证梯度下降能找到，即使找到了也不保证稳定。这才是GAN难训练的数学根源，不是什么工程问题。

三大死法

1. Mode Collapse：生成器开始摸鱼

Mode collapse 的表现是生成器只输出少数几种结果，不管你给什么噪声输入，生成的东西都差不多。在图像生成里可能是只生成某几个角度的人脸，在时间序列里可能是只生成几种固定模式的曲线。

原因很清楚：生成器发现，只要固定输出某几个”判别器暂时认不出来”的样本，loss 就能一直低着，于是它就这么干了。

解法：用 Wasserstein 距离替换 JS 散度。

标准 GAN 用的是 Jensen-Shannon 散度来衡量真实分布和生成分布的距离，但 JS 散度在两个分布没有重叠时是个常数（$\log 2$），梯度直接消失。WGAN 改用 Wasserstein-1 距离：

$$W(p_\text{data}, p_g) = \sup_{|f|L \leq 1} \mathbb{E}{x \sim p_\text{data}}[f(x)] - \mathbb{E}_{x \sim p_g}[f(x)]$$

其中 $|f|_L \leq 1$ 是 1-Lipschitz 约束。这个距离在分布不重叠时也有意义的梯度，mode collapse 问题能显著缓解。

更进一步，RpGAN（Relativistic paired GAN）[Jolicoeur-Martineau, 2018] 让判别器不再只判断”这个样本是真是假”，而是判断”真实样本比生成样本更真实的程度”，信号更稳定。

梯度惩罚方面，R1 正则化 [Mescheder et al., 2018] 在实践中非常有效：

$$R_1 = \frac{\gamma}{2} \mathbb{E}{x \sim p\text{data}}[|\nabla_x D(x)|^2]$$

只惩罚真实数据点上的梯度，不像 WGAN-GP 那样需要对插值点采样，计算更干净。

2. 训练振荡：loss 在抖，什么都没学

这个症状是 D 和 G 的 loss 都在一个范围里反复横跳，看起来”训练中”但实际上两个网络在互相对抗导致参数一直震荡。

解法：

  • Fixup 初始化：不用 BatchNorm，改用 Fixup [Zhang et al., 2019] 的残差网络初始化方案，让梯度信号更干净地传播
  • EMA（指数移动平均）：维护一份生成器权重的 EMA 副本用于推理，可以平滑掉训练过程中的震荡
  • 学习率调度：生成器和判别器用不同的学习率，通常判别器用更小的学习率，别让它跑太快

3. 判别器过强：生成器根本没得学

如果判别器太强，它能轻松识别所有生成样本，那么 $D(G(z)) \approx 0$，$\log(1 - D(G(z))) \approx 0$，生成器的梯度消失，它学不到任何东西。

解法是 Relativistic GAN [Jolicoeur-Martineau, 2018]，判别器的目标从”这是真/假”变成”真实样本比生成样本真实多少”，生成器总能从这个相对差距里得到有意义的梯度。

渐进式训练（Progressive Growing）[Karras et al., 2018] 从低分辨率开始训，逐步加大，也能让双方能力比较均衡地成长。

工程 Checklist

训练时要盯的东西：

  • D loss 持续下降趋近 0 → 判别器赢了，生成器没在学
  • G loss 持续下降但 D loss 不动 → mode collapse 早期信号
  • FID / 实际样本质量每隔几百步检查一次，不要只看 loss
  • 梯度范数：D 和 G 的梯度范数都要监控，太小（消失）太大（爆炸）都要警惕

判别器强度平衡：

  • D 的更新步数和 G 保持 1:1 或者 2:1，不要让 D 更新太多
  • R1 梯度惩罚系数 $\gamma$ 从 10 开始调，太大会拖慢 G 的学习

时间序列的特殊性

做时间序列 GAN 有一个经典问题：序列是 1D 的，但直接用 1D 卷积判别器效果不好，因为局部的时间相关性很难被捕捉到。

MTSIR3-GAN 里用的策略是把时间序列切成小片段，变成 2D 的 patch 输入判别器。这个思路来自 PatchGAN [Isola et al., 2017]，原来用在图像上，但对时间序列同样有效——patch 判别器只需要判断”这段局部时间模式是否真实”，而不是对整段序列打分，梯度信号更局部也更稳定。

实际效果：对比直接用全序列判别器，patch 策略在插补任务上 MAE 能下降 8-12 个点，不是小数字。

美琴和咲夜的博弈

快速类比：生成器像御坂美琴，判别器像十六夜咲夜。美琴一直在想怎么放出判别器看不穿的电击（生成逼真样本），咲夜用精确的时间掌控来识别哪些不对劲（判断真假）。Mode collapse 是美琴发现只用一招电磁炮就能骗过咲夜，然后再也不练别的了。

训练好的 GAN 是两人都没摆烂、都练到了极限的状态。这个状态很难达到，但存在。

最后说一句

GAN 在 2022-2024 年因为 Diffusion 的崛起被唱衰了很多，但在条件生成、时间序列、低延迟推理这些场景里，GAN 的速度优势还是实实在在的。Diffusion 推理要走几十步，GAN 一步出结果，这个差距在实时系统里不是细节是命门。

所以训练 GAN 这个技能现在还值得会。踩坑之后你会发现那篇 WGAN 论文里每一个细节都是血写的。
